Cross-referencing in Hugo

Jul 31, 2018 in DEV
dev hugo cross-reference link anchor slug
4 min read

I have been spending quite a fair amount of time working with Hugo, especially on building this blog. Since then Hugo keeps amazing me with a plethora of nice features, ranging from systematic content structures, well-organised theming, to powerful templating, several useful functions and shortcodes.

The documentation is quite rich and covers most of Hugo major aspects. Nonetheless, it is sometimes caught off-speed due to the incredible development progress made by amazing Hugo authors and contributors ❤️.

The most recent missing (or hidden) part is “cross-referencing” that costed me few hours searching around and experimenting before coming up with a reasonable solution. I hope that this post would save you that effort.

We will consider a simple workflow of Hugo generator illustrated below. Based on that, we will discuss the relevant cross-referencing concepts supported by HTML and Markdown and how Hugo brings them together.

+-----------+         +------+          +--------+
| Markdown  | +-----> | Hugo | +------> |  HTML  |
+-----------+         +------+          +--------+

Basic HTML Concepts

A link is one of the basic constructs of HTML—the core language of the Web and also the main resources generated by Hugo. It represents the connection between two Web resources. The most popular kind of links are hyperlinks that are exposed to end-users and can be used to navigate to the linked resources.

We often see a hyperlink created by the HTML <a> element (or anchor element). A simple <a> element can be defined with the following reduced syntax.

<!-- reduce syntax -->
<a href="URL" target=[_self|_blank_parent|_top] >Linked text</a>

A URL is a type of universal identification of a Web resource. Simply put, a URL has the form of [scheme:]path[?query][#fragment]:

  • scheme = ftp | file | gopher | http | https | ws | wss
  • path: the path to the resource, such as www.example.org/index.html
  • query: (optional) used as inputs for dynamic Web page processing (which is not the case here as we mainly use static pages)
  • fragment: (optional) used for further processing, e.g. anchoring.

Please refer here for more detailed URL specification.

For instance, this HTML code.

<!-- link to a URL -->
<a href="https://gohugo.io/">Hugo -- the fastest static-site generator</a>

will show a phrase “Hugo – the fastest static-site generator” linked to https://gohugo.io. Clicking (on desktop) or touching (on mobiles) that phrase will bring us to Hugo home page.

We can also create a link navigating to an element inside an HTML resource. To do this, we must specify a URL fragment that contains a # symbol.

<!-- link to a URL fragment -->
<a href="https://gohugo.io/getting-started/installing/#quick-install">Go to Hugo's "Quick Install" part</a>

The fragment #quick-install is often called a named anchor. There are two ways to define the aforementioned anchor part. Before HTML 5, we can use <a name="quick-install"></a>. But the attribute name of <a> is no longer valid in HTML 5 (but may still work in some Web browsers).

The second method, which is valid for both HTML 4 and 5, is to use id="quick-install" attached to the corresponding HTML element, for instance, <h2 id="quick-install">. Note that the HTMLs generated by Hugo are using this anchoring method.

Note that, if the part before # is absent, then the <a> element will link to a named anchor in the current document.

Cross-Referencing in Markdown

I mainly use Markdown contents in my Hugo projects. Thus, this post will focus merely on cross-references in Markdown.

Markdown’s syntax for establishing links is simpler and less verbose than HTML’s.

[Linked text](URL)

As such, the two hyperlinks in the previous section can be defined equivalently in Markdown code.

[Hugo -- the fastest static-site generator](https://gohugo.io/)

[Go to Hugo's "Quick Install"](https://gohugo.io/getting-started/installing/#quick-install)

It is interesting that the URL part can reference to another Markdown file. That means we can establish the connections among Markdown contents. This feature is extremely useful as we just concentrate on writing and linking Markdown without caring much about the generated HTMLs.
