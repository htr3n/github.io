CSS preprocessors: SASS vs. LESS

Jul 6, 2018 in DEV
dev web css less sass scss
3 min read

A preprocessor is type of data processor component that takes input data and produces some output data according to some predefined or configured semantics and logics. As the name implies, preprocessing stage often happen beforehand. For instance, C preprocessor is often used to handle the C header files, macros, and so on, before the actual compilation.

CSS (Cascading Style Sheet) is a declarative language used for defining the presentation of HTML documents. It was a nice idea to separate data (HTML content) with data formatting (CSS). Nonetheless, developers often find defining CSS elements repetitive and redudant. That’s why CSS preprocessors like SASS, LESS, etc. were developed to extend CSS with extra handy concepts and structures such as variables, functions, mixins, and so forth. A CSS preprocessor will transform code in the corresponding languages into CSSs. Some can also perform extra tasks such as compressing or link rewriting. Web developers have some languages to adopt such as SASS (Syntactically Awesome Stylesheets), LESS, and Stylus.

Sass was designed and developed in 2006 mainly using Ruby. It provides nice concepts like variables, loops, nesting, mixins, and inheritance. The original indented SASS syntax inherited from Haml was closer to Python than CSS, and hence, got complaints from Web developers. Less.js (or Less) inspired by Sass aims to get closer to Web developers with a more CSS alike syntax and JavaScript implementation but it provides a smaller set of features (therefore the name?). Nevertheless, Sass got influenced back and created SCSS (dubbed “Sassy CSS”) as a superset of CSS3. Stylus is very similar to Less with more compact syntax and a strong competitor to Sass with rich extension libraries.

Per some analyses, Sass and Stylus lean towards imperative/iterative language structures (e.g. if, loop) whilst Less seemingly adopts more declarative and functional concepts (e.g. loops defined with recursions and conditional branching with guard conditions).

In the subsequent parts, let’s walk through some fundamental aspects of these CSS preprocessors.

Variables

To declare a variable, Sass uses $var : value , Less uses @var : value, Stylus does not use prefix but equal sign var = value.

Note that a variable might associate with a scope of validity.

$body-color: #303030;
body {
	  $content-margin: 1em;
    color: $body-color;
    margin: $content-margin;
}

The variable $body-color is available for most of rules whilst $content-margin is only valid within the block of body.

Variable interpolation (substitution)

Sass, as based on Ruby, uses #{} for substitution of variables in an expression. This can be used to dynamically handling elements based on the actual values of the substituted variables.

$my-class : featured-image ;
section .#{my-class} {
  color: green;
}

Data types

Sass supports essential data types for CSS declarations, including numbers (e.g. 10, 1em, 30px), strings (e.g."Fira Sans"), colors (e.g. #303030, rgb(123, 123, 123)), boolean (e.g. true, false), comma or space separated values (e.g. 1px 2px 3px, 'Source Code Pro', monospace).

Sass also provides basic operators such as addition, subtraction, multiplication, and division to form expressions of variables.

Nesting

Instead of repeat several selectors to define their rules like CSS, Sass enables nested selectors or properties that significantly shorten the code.

// nested selectors
.content {
  p, section {
    padding: 5px;
  }
}
// nested properties 
h1 {
  font: {
    family: sans-serif;
    size: 2em;
    weight: bold;
  }
}

Referencing Parent Selector

From a nested child rule, we can reference to the intermediate parent using &.

a {
  text-decoration: none;
  & : hover {
    text-decoration: underline;
  }
}

LESS

Like and influenced by Sass, Less enhances CSS with mixins, variables, and nested style rules. Its simplicity draws lots of attention from the Web development community.
