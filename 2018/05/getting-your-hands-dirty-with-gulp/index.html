Getting your hands dirty with Gulp

May 1, 2018 in DEV
gulp javascript node.js npm yarn
2 min read

Gulp is a JavaScript/Node.js based tool often used for automating many development tasks such as analysing, compiling, compressing, transforming source code and application resources, to name but a few. It is very popular alongside Grunt, and Webpack. Among those, Gulp and Grunt are referred as task runners (defining based on tasks) whilst Webpack is more of a module bundler (defining based on modules, inputs, outputs).

Gulp leverages Node’s streaming API for establishing pipelines of input and output data. Whilst streaming can potentially reduce disk I/O cost for better performance, it also requires Gulp plugins to implement compatible streaming interfaces.

Note

Gulp 4.0+ release contains some breaking changes to earlier versions. We will dig into this matter in the later part. For now, we can assume the syntax and commands of the most popular Gulp 3.x.

Getting Started with Gulp 3

Let us take a simple Gulpfile.js that compiles an SCSS file to CSS as following.

var gulp = require('gulp');
var sass = require('gulp-sass');

var sassSourceFile = 'styles/scss/main.scss';
var watchedResources = 'styles/scss/**/*';
var outputFolder = 'themes/hyde-hyde/static/css';

gulp.task('scss', function () {
  gulp.src(sassSourceFile)
    .pipe(sass().on('error', sass.errorLog)
    .pipe(gulp.dest(outputFolder));
});
Essential Gulp 3’s built-in constructs
  • gulp.task(name, fn): used for defining tasks
  • gulp.src(globs [, options]): used to specify input resources
  • gulp.watch(globs [, options] [, fn]): used to monitoring changes in application resources
  • gulp.dest(path [, options]): used to specify the output destination of the processed resources
  • .pipe(function): used to chain plugins or applications together
  • gulp.parallel(...tasks) and gulp.series(...tasks): (since 4.0)
